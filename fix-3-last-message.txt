Explicit limitations and follow-ups:
- browser page ax uses a bounded DOM/ARIA accessibility-style fallback, not a full native platform accessibility dump. It stores sanitized short-session refs under EFP_BROWSER_HOME and refs should be regenerated after navigation or DOM changes.
- Ref-based actions support the selected page execution context. Frame-specific refs are rejected for actions in this pass; use browser frame list and browser frame snapshot for frame reads.
- browser network start/list/wait/stop/clear records sanitized HAR-lite metadata only after start injects the page-side recorder. It captures fetch/XHR metadata and resource timing metadata, but never headers, cookies, storage, request bodies, or response bodies.
- browser page console/errors capture console/runtime diagnostics only after recorder injection. Messages and stacks are redacted/truncated, and object previews are not returned.
- --pierce traverses open shadow roots only. Closed shadow roots are intentionally inaccessible.
